## 1. 状態管理（State）

### 状態定義

| 状態変数 | 型 | 説明 | 初期値 | ログアウト時 |
|---------|-----|------|-------|-----------|
| `session_auth` | boolean | ログイン中か否か | `false` | `false` に遷移 |
| `progress_stage` | number (0..5) | 進行度レベル | `0` | **保持される** |

### progress_stage のレベル定義

| stage | 名称 | 条件 | 表示状態 |
|-------|------|------|--------|
| 0 | 未ログイン | ログイン前 | 通常表示（痕跡なし） |
| 1 | ログイン直後 | ログイン完了、Case02未実施 | 微差（痕跡1～2個） |
| 2 | Case02クリア後 | Case02コード正解 | 痕跡増加（復号材料露出） |
| 3 | Case03クリア後 | Case03コード正解 | 復号がより明確に |
| 4 | Case04クリア後 | Case04コード正解 | 露骨な異常（統合直前） |
| 5 | 最終完了 | Case05コード正解 | 観測完了演出 |

### 表示パターン（2軸）

```
【表示決定マトリックス】

通常表示（session_auth = false）
  └─ progress_stage で「痕跡の強度」を変化
  
管理者表示（session_auth = true）
  └─ progress_stage で「復号材料の露出」を変化
```

## 3. UI導線設計

### グローバルナビゲーション（全ページ共通）

```
┌─────────────────────────────────────────┐
│ ホーム  │  掲示板  │  日記  │  リンク  │
└─────────────────────────────────────────┘
```

**ルール**:
- auth状態に関わらず同じナビを表示
- ログイン中でもナビは増やさない（「壊れて露出」する設計で統一感を出す）

### ログアウト導線

**配置**: `/admin/terminal` の最下部に小さく配置

```
─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
[ ログアウト ]
```

**ルール**:
- 通常ページ側にはログアウトボタンを置かない
- ログアウトは「管理者行為」として `/admin` エリアに閉じ込める

### 比較用の「痕跡」導線

**対象**: ログアウト後で `progress_stage >= 1` の場合

**表示例**（フッター）:
```
最後の管理者アクセス: 2026-02-28 14:23
```

または、カウンター横に小さな異常（例：桁が1つ欠ける）を挿入。

**ルール**:
- 通常ページは基本「ほぼ元通り」に戻す（比較が成立する前提）
- 痕跡は「比較で気づく」程度の控えめさを維持

## 5. 入力正規化ルール（採点仕様）

### 受け入れ・拒否条件

| 項目 | 仕様 |
|------|------|
| **受け入れ対象** | ひらがな・カタカナ・漢字・全角句読点のみ |
| **拒否対象** | 半角英数・半角記号・改行・Tab |
| **空白** | 全角スペースも含めて削除 |

### 採点前の正規化処理

以下の順序で自動正規化を実行：

```
1. trim()：前後の全角スペース・改行を削除

2. ひらがな統一：カタカナ → ひらがなに自動変換
   例：「シンダイ」 → 「しんだい」
   ※ 平文字基準（ぁぃぅぇぉ小文字は保持）

3. 長音記号統一：「ー」「−」「―」 → 「ー」に統一

4. 濁点結合正規化：「は゛」「か゛」など → 「ぱ」「が」に統一
   ※ Unicode正規化（NFD→NFC）で実装

5. 小書き仮名検証：「ぁぃぅぇぉ」「ゃゅょ」は本来の形で判定

6. 旧字体・異体字：検出時は拒否
   エラー文：「旧字体・異体字には対応していません」
```

### 判定フロー

```
入力
  ↓
[受け入れ可能な文字のみ？]
  ├─ NO → エラー「半角英数は使えません」
  └─ YES
      ↓
    [正規化処理を実行]
      ↓
    [正規化後の文字列で完全一致判定]
      ├─ 一致 → 【正解】→ stage進行
      └─ 不一致 → 【不正解】→ エラー文言
```

### 採点例

| 入力 | 正規化後 | 判定 | 理由 |
|------|---------|------|------|
| `サカサマノハシ` | `さかさまのはし` | ✅ 正解 | ひらがな統一で一致 |
| `坂逆間の橋` | 拒否 | ❌ | 漢字混じりの形式が違う |
| `さかさまの橋` | `さかさまのはし` | ✅ 正解 | カタカナ→ひらがな統一 |
| `シンダイ` | `しんだい` | ✅ 正解（応募内容による） | カタカナ統一 |
| `sa-ka-sa-ma` | 拒否 | ❌ | 半角英数は拒否 |

## 6. 置換表（全角版）

### 置換表 R1：五十音ベース

**原理**: 1対1の全音置換（ひらがな → ひらがんに置換）

```
【置換表 R1 完全版】

あ ↔ く      い ↔ や      う ↔ め      え ↔ こ      お ↔ ね
か ↔ ら      き ↔ ち      く ↔ と      け ↔ さ      こ ↔ た

さ ↔ あ      し ↔ か      す ↔ き      せ ↔ う      そ ↔ は
た ↔ そ      ち ↔ よ      つ ↔ な      て ↔ り      と ↔ ろ

な ↔ の      に ↔ に      ぬ ↔ ぬ      ね ↔ お      の ↔ ひ
は ↔ ん      ひ ↔ ぼ      ふ ↔ も      へ ↔ し      ほ ↔ ふ

ま ↔ み      み ↔ え      む ↔ む      め ↔ ほ      も ↔ へ

や ↔ ま      ゆ ↔ ゆ      よ ↔ ち
ら ↔ か      り ↔ せ      る ↔ つ      れ ↔ け      ろ ↔ て

わ ↔ さ      を ↔ や      ん ↔ は
```

**特別ルール**:
- 濁点・半濁点は置換前後で維持
  - 例：「が」置換時は「ぐ」の置換先に濁点付与
  - ただし実装簡略化のため「が」→「ぐ」など独立した文字として扱う方が安全
- 小書き仮名（ぁぃぅぇぉ等）は置換しない（出現しない仕様）

### 段階的な露出計画

| stage | ページ | 提示内容 | 詳細 |
|-------|--------|--------|------|
| 1 | /admin | **なし** | 第1暗号（合言葉）のみ |
| 2 | /bbs（固定スレ） | **対照表の断片** | あ→く、さ→あ、ん→は（3行） |
| 2 | /links | **欠損ヒント** | 説明文に「置換…された…」と記載 |
| 3 | /bbs/rules | **対照表の拡張** | 半分程度（約25文字分） |
| 4 | /diary / 本文 | **復号材料として使用** | 本文の一部が置換される |
| 5 | /case/05 | **不要** | すべて復号済み |

### 復号方法（プレイヤー向け）

復号はシンプルな置換：

```
例1：「ぐぐぐだび」を復号
あ→く なら 逆に く→あ で復号
「ぐぐぐだび」→「あああだひ」？ （内容が変だ）
違うルールを試す...

例2：置換表から「さ→あ」を見つけたら
「最後」と読みたい場合→「妻後」を試す
（漢字は置換されないため、想像で補完）
```

**「漢字は置換されない」という仕様が復号難度を適切に保つ**

## 7. 4要素統合のテンプレ

### 統合方式：「四字熟語」生成

各Case（謎）で「**漢字1文字**」を得て、並び替えて最終答えを形成。

### 各Case の「得られる文字」

| Case | 取得文字 | 出典・示唆 | 意味合い |
|------|---------|----------|--------|
| Case02 | **動** | 「動く庭」より | 動きがある / 不安定 |
| Case03 | **静** | 「静寂の橋」より | 静止 / 沈黙 |
| Case04 | **輪** | 「輪廻の鐘」より | 循環 / 繰り返し |
| Case05 | **廻** | 統合試練で判明 | 廻転 / 回帰 |

### 最終答え：「動静輪廻」（どうせいりんね）

**特性**:
- 実在する4字熟語類似（輪廻転生の「輪廻」を含む）
- 読みで全てひらがな化可能：「どうせいりんね」
- プレイヤーが「漢字4文字」を入力 → 採点時にひらがなに正規化で判定

**採点例**:
```
入力：「動静輪廻」
正規化：「どうせいりんね」
判定：✅ 正解

入力：「ドウセイリンネ」
正規化：「どうせいりんね」
判定：✅ 正解
```

## 8. 設計上の制約・ルール

### 比較ギミック（入れすぎ防止）

- **比較必須のギミック**: 最大1回まで
  - 例：掲示板の固定スレだけ（auth時のみ見える）
- **それ以外**: 「比較すると気づける追加の手がかり」程度に留める
- **理由**: プレイヤーの迷子防止

### ログアウトでしか見えない情報

- **ルール**: 作らない
- **理由**: 理不尽防止＆詰み防止

### エラー・ヒントメッセージ

- **ルール**: 不正解回数でメッセージが侵食しても、ヒント削減はしない
- **理由**: ゲームの詰みを防ぐ（難度調整はメッセージ文言で行う）
